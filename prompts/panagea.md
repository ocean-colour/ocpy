#### Overview

PANAGEA (the Valente et al. 2022 "version 3" compilation, PANGAEA
doi:10.1594/PANGAEA.941318) is a global compilation of in situ bio-optical
data assembled to validate ocean-colour satellite products for the ESA
Ocean Colour CCI / OMAPS projects. It merges many archives (MOBY, BOUSSOLE,
AERONET-OC, SeaBASS, NOMAD, MERMAID, AMT, ICES, HOT, GeP&CO, …), spans
1997-01-02 to 2021-09-27, and is global in coverage (lat −78.04°..89.97°,
lon −180°..180°). License: CC-BY-4.0.

#### Local data

Located at `$OS_COLOR/PANAGEA/V3/`
(= `/home/xavier/Projects/Oceanography/data/Color/PANAGEA/V3/`):

- `summary.txt` — the collection-level PANGAEA metadata (citation, abstract,
  coverage, and the list of 7 child datasets with their DOIs).
- `datasets/` — the 7 `.tab` data files.

#### The 7 `.tab` files

All files share the same layout: a PANGAEA header block delimited by
`/* ... */`, then a single tab-separated **column-name line**, then
tab-separated data. The header block lists every column under
`Parameter(s):`, one line per column, in order. Missing values are encoded
as **empty fields** (pandas reads them as NaN). All files share a common
integer key column `ID` (idx).

| File | Content | Header ends (line) | Col line | # cols | ~# data rows |
|------|---------|-------------------:|---------:|-------:|-------------:|
| `insitudb_chla_V3.tab` | Chlorophyll-a (HPLC + fluorometric/spectro) | 28 | 29 | 15 | 82,543 |
| `insitudb_rrs_V3.tab` | Remote-sensing reflectance Rrs, native wavelengths | 969 | 970 | 956 | 68,641 |
| `insitudb_rrs_satbands2_V3.tab` | Rrs aggregated within ±2 nm of satellite bands | 159 | 160 | 146 | 68,641 |
| `insitudb_rrs_satbands6_V3.tab` | Rrs aggregated within ±6 nm of satellite bands | 159 | 160 | 146 | 68,641 |

| `insitudb_iopskdtsm_satbands6_V3.tab` | Same IOPs/Kd within ±6 nm of satellite bands | 481 | 482 | 468 | 7,422 |

#### Column conventions

- **Common metadata columns** (first four of every file): `ID (idx)`,
  `Date/Time` (ISO 8601, e.g. `1997-01-02T12:48`), `Latitude`, `Longitude`.
  Most files also carry `Depth water [m]` (from ETOPO1, clamped to ≤0) and a
  set of provenance `Comment` columns (dataset / subdataset / contributor)
  plus quality flags (`QF time`, `QF Chl`).
- **Native-wavelength files** name spectral columns like
  `RRS [1/sr] (at 313nm)` or `APH [1/m] (at 290.2nm)` — hundreds of
  closely-spaced wavelengths (hence the very wide IOP/Rrs files).
- **Sat-band files** name columns in pairs: the value at a sensor band, e.g.
  `RRS [1/sr] (at +-2nm of OLCI-S3A band 11)`, followed by the actual
  `Lambda [nm] (assigned to ... band 11)`. Sensors covered: SeaWiFS,
  MODIS-AQUA, VIIRS, OLCI (S3A/S3B) and MERIS.
- **IOP variables** in `iopskdtsm`: `APH` (algal pigment absorption),
  `acCDOM` (CDOM + detrital absorption), `bbp` (particulate backscatter),
  `KD` (diffuse attenuation), `TSS` (total suspended matter / TSM).

#### How to load (verified in `ocean14`)

```python
import pandas as pd
df = pd.read_csv(path, sep="\t", skiprows=N)   # N = header-end line number
```

The number of `skiprows` equals the "Header ends" line in the table above
(the line containing the closing `*/`), so the next line — the column
header — becomes the DataFrame header. Verified loading `chla` (82,543 ×
15) and a sat-band Rrs file cleanly; empty fields → NaN, `ID`/`Depth` are
int, lat/lon/values are float.

#### Joining across files

`ID` is a global observation index shared across all 7 files (it is not
contiguous within a file — it ranges 1..151,673 with gaps). The same `ID`
in two files refers to the same in-situ observation, so files can be merged
on `ID`. Overlap is partial (e.g. chla ∩ rrs_satbands2 = 3,840 IDs), since
not every observation has every measurement type.

#### Implications for the API

- A single loader can serve all 7 files: skip the right number of header
  lines (or detect the `*/` line automatically), then parse the column line.
- The header `Parameter(s):` block is a machine-readable column dictionary
  (long name, unit, PI, method, comment) that could be parsed to build
  tidy/renamed columns and per-column metadata.
- For the wide spectral files, a "wide → long" reshape (ID, wavelength,
  value) is likely the most useful in-memory form for downstream analysis.

#### Resolved design decisions (from Q&A, 2026-06-15)

These reflect the user's answers and now drive the plan:

- **All 7 files.** One generic loader handles every `.tab` file.
- **Auto-detect the header.** The loader scans for the line ending the
  `/* ... */` block (the `*/` line) and treats the following line as the
  column header — no hard-coded `skiprows` per file. This survives any
  re-download.
- **Primary use case = generating individual Rrs (and IOP) spectra.** The
  driving activity is: given an observation, produce its spectrum — a set of
  (wavelength, value) pairs. So the design is:
  1. Load each file faithfully into a wide DataFrame indexed by `ID`.
  2. Parse the wavelength out of each spectral column name (e.g.
     `RRS [1/sr] (at 349.01nm)` → `349.01` as a float; sat-band columns
     labelled by their paired `Lambda [nm]` value).
  3. Provide a helper that, for a given `ID`, returns that observation's
     spectrum as a wavelength-indexed `pandas.Series` with missing
     wavelengths dropped — i.e. the natural input for plotting/fitting a
     single Rrs spectrum. A bulk long-format reshape (ID × wavelength ×
     value) is offered as a secondary convenience for cross-observation
     analysis.
  This keeps the wide table available (faithful, fast `ID` lookup) while
  making single-spectrum extraction the easy path.

#### 4. Generic loader `load(key)` (Q2: auto-detect header)

Steps, all implemented as small helper methods:

1. `_find_header_end(path)` — scan lines for the one equal to `*/`, return
   its line number; the next line is the column header. No hard-coded
   `skiprows`. (Verified: this matches the 7 files' header-end lines.)
2. `pandas.read_csv(path, sep="\t", skiprows=n)` to get the raw wide frame.
3. Set the index to the `ID` column (`ID (idx)` / `ID`) so observations are
   addressable by their global id.
4. Parse `Date/Time` to datetime; keep `Latitude`, `Longitude`,
   `Depth water` as numeric.
5. Rename columns to friendly names and attach per-column metadata
   (sections 5–7).
6. Stash a `kind` and `wavelengths` summary in `df.attrs` for downstream use.

#### 5. Column naming (Q5: option b — friendly names, originals kept)

- Metadata columns → clean snake/short names: `date_time`, `lat`, `lon`,
  `depth_m`, provenance `dataset` / `subdataset` / `contributor`, quality
  flags `qf_time`, `qf_chl`.
- For files whose provenance is method-specific (chla has hplc *and* fluor
  provenance), names are prefixed: `hplc_dataset`, `fluor_contributor`, etc.
- Spectral columns → see section 6.
- The **original** PANGAEA column name, long name, unit, PI, method and
  parsed attributes are preserved in a per-column metadata dict, exposed via
  `column_metadata(df)` and stored in `df.attrs["columns"]`. Nothing from the
  file is lost.

#### 6. Spectral columns & wavelength parsing (Q3 + Q6)

Primary use case = **generate individual Rrs/IOP spectra**. Design:

- **Native-wavelength files** (`rrs`, `iop`): parse the wavelength out of each
  spectral column name, e.g. `RRS [1/sr] (at 349.01nm)` → `349.01` (float).
  Friendly column form keeps the variable + wavelength, e.g. `rrs_349.01`,
  and the float wavelength is recorded in metadata.
- **Sat-band files** (`*_sat2/6`): value columns are paired with a
  `Lambda [nm] (...)` column. Per Q6, the **per-row `Lambda` value is the
  wavelength**, and the sensor/band (e.g. `OLCI-S3A band 11`) is carried as a
  label in metadata. So spectra are built from (Lambda value, RRS value)
  pairs rather than the nominal band number.
- **IOP variables** kept as distinct families: `aph`, `acdom`, `bbp`, `kd`,
  plus scalar `tss`. Per Q7, quantities measured by multiple methods are kept
  separate (chla → `chla_hplc`, `chla_fluor`; never silently merged).

`spectrum(df, obs_id, kind="rrs")` returns a `pandas.Series` indexed by
wavelength (nm, float), values for that one `obs_id`, with missing
wavelengths dropped — the natural input for plotting/fitting a single
spectrum. `kind` selects the variable family (`rrs`, `aph`, `acdom`, `bbp`,
`kd`). For sat-band frames it uses the paired `Lambda` columns for the index.

#### 7. Bulk reshape `to_long(df, kind)`

Secondary convenience: return a long/tidy frame
`[ID, wavelength, value, (sensor, band)]` for cross-observation analysis,
built by melting the selected variable family. Useful for stats across many
observations; the per-ID `spectrum()` remains the easy path for single
spectra.

#### 8. Metadata access

- `column_metadata(df)` → `{friendly_name: {orig, long_name, unit, pi,
  method, comment, wavelength, sensor, band, variable}}`.
- Could optionally also parse the file's `Parameter(s):` header block to
